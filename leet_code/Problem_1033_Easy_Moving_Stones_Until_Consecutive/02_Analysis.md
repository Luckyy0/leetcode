Bài toán này thuần túy về Toán học đếm số kẽ hở (Khoảng cách). Gọi a, b, c là 3 vị trí đã được **Sắp xếp Tăng dần** (x < y < z).
- Nếu 3 đá đã dính trọn vào nhau (`x, y, z` cách đều nhau 1 đơn vị, vd: 2,3,4): Số bước MAX = 0, MIN = 0.
- Số bước TỐI ĐA (Maximum Moves): Cứ nhích từng bước rùa bò 1 đơn vị lấp kín trọn thung lũng khoảng cách 2 bên viên đá Giữa `y` -> `Max = (y - x - 1) + (z - y - 1)`.
- Số bước TỐI THIỂU (Minimum Moves): 
   + Nếu viên đá ở giữa đã nằm tì hằn (cách 1 khoảng trống) với hai đứa đầu đuôi (vd khoảng cách = 2 như 4, 6) -> Chỉ cần ném mút Rìa vào chính cái Lỗ hổng Trống Rỗng đó LÀ XONG PHIM (1 BƯỚC).
   + Những thế trận cách xa vợi vợi khác, luôn dễ dàng Nhặt cục ngoài rìa Ném Áp Sát Vào Cụm Trong Kín. Trầm kha nhất cũng chỉ nhấc tay thả đúng `2 BOOCK` GÓC Ném Là Khớp trọn cả Lỗ!

### Java Optimal Implementation Let
### *Triển khai giải pháp Java*

```java
import java.util.Arrays;

class Solution {
    public int[] numMovesStones(int a, int b, int c) {
        int[] pos = {a, b, c};
        Arrays.sort(pos);
        
        int x = pos[0];
        int y = pos[1];
        int z = pos[2];
        
        int maxMoves = (y - x - 1) + (z - y - 1); // Tổng Độ Rỗng
        int minMoves = 2; // Default Ám chỉ Trượt Góc Trái Tới Gần, Góc Phải Dán Kín Là Mắc Cỡ Nhất (Tối Thiểu Max 2)
        
        if (z - x == 2) {
            // (1,2,3) -> Liền kề cứng nén
            minMoves = 0;
        } else if (y - x <= 2 || z - y <= 2) {
            // Kẽ nứt chỉ đúng 1 lỗ nhỏ, Thả Đá Khớp Điền Nhanh Gọn lẹ (1 Phát Ăn Ngay)
            minMoves = 1;
        }

        return new int[]{minMoves, maxMoves};
    }
}
```
