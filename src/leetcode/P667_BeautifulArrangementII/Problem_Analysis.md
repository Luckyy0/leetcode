# 667. Beautiful Arrangement II / Sắp xếp Đẹp II

## Problem Description / Mô tả bài toán
Given two integers `n` and `k`, you need to construct a list which contains `n` different positive integers ranging from 1 to `n` and obeys the following requirement:
Cho hai số nguyên `n` và `k`, bạn cần xây dựng một danh sách chứa `n` số nguyên dương khác nhau từ 1 đến `n` và tuân theo yêu cầu sau:

Suppose this list is `a1, a2, a3, ..., an`, then the list `|a1 - a2|, |a2 - a3|, |a3 - a4|, ..., |an-1 - an|` has exactly `k` distinct integers.
Giả sử danh sách này là `a1, a2, a3, ..., an`, thì danh sách `|a1 - a2|, |a2 - a3|, |a3 - a4|, ..., |an-1 - an|` có chính xác `k` số nguyên khác nhau.

Return the list. If there are multiple answers, return any of them.
Trả về danh sách. Nếu có nhiều câu trả lời, hãy trả về bất kỳ câu trả lời nào trong số đó.

---

## Theoretical Foundation / Cơ sơ lý thuyết

### Interleaving Pattern / Mô hình Xen kẽ
To get `k` distinct differences, we can interleave numbers from the two ends of the range.
Để có được `k` sự khác biệt riêng biệt, chúng ta có thể xen kẽ các số từ hai đầu của phạm vi.

For a range `[left, right]`:
- Difference `k, k-1, k-2, ..., 1` can be generated by alternating between picking the smallest and largest available numbers.
- Once we have `k` differences, the remaining `n - (k+1)` numbers can be arranged in ascending order to contribute a difference of 1.

### Complexity / Độ phức tạp
- **Time**: O(N) where N is the given count.
- **Space**: O(N) to store the result.

---

## Analysis / Phân tích

### Approach: Ping-pong Construction
Fill the first `k+1` elements by picking from `1, n, 2, n-1, ...`. Then fill the rest sequentially. This ensures exactly `k` differences: `(n-1), (n-2), ... , (n-k)`... actually it's easier to use a smaller range.
Điền vào `k+1` phần tử đầu tiên bằng cách chọn từ `1, n, 2, n-1, ...`. Sau đó điền nốt phần còn lại theo thứ tự. Điều này đảm bảo có chính xác `k` sự khác biệt.

Correct logic:
Elements 1 to `n-k-1` are `1, 2, 3...`.
Then alternate: `n-k, n, n-k+1, n-1, ...`
Actually, even simpler:
First `n-k` elements: `1, 2, ..., n-k`.
Remaining `k` elements alternate high/low.
