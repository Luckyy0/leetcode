# 264. Ugly Number II / Số Xấu II

## Problem Description / Mô tả bài toán
An **ugly number** is a positive integer whose prime factors are limited to `2`, `3`, and `5`.
Một **số xấu** là một số nguyên dương có các thừa số nguyên tố chỉ giới hạn trong `2`, `3`, và `5`.

Given an integer `n`, return the `n`-th **ugly number**.
Cho một số nguyên `n`, hãy trả về **số xấu** thứ `n`.

### Example 1:
```text
Input: n = 10
Output: 12
Explanation: [1, 2, 3, 4, 5, 6, 8, 9, 10, 12] is the sequence of the first 10 ugly numbers.
```

### Example 2:
```text
Input: n = 1
Output: 1
```

## Constraints / Ràng buộc
- `1 <= n <= 1690`

---

## Theoretical Foundation / Cơ sở lý thuyết

### Dynamic Programming (Three Pointers) / Quy Hoạch Động (Ba Con Trỏ)
We want to generate the ugly numbers in ascending order.
The first ugly number is `1`.
Any subsequent ugly number is generated by multiplying a previous ugly number by 2, 3, or 5.
`next_ugly = min(ugly[p2] * 2, ugly[p3] * 3, ugly[p5] * 5)`

- We keep three pointers: `p2`, `p3`, `p5`.
  - `p2` points to the index of the ugly number that, when multiplied by 2, gives the next candidate from the "x2" stream.
  - `p3` for "x3" stream.
  - `p5` for "x5" stream.
- Initially `p2 = p3 = p5 = 0` (pointing to `ugly[0] = 1`).
- In each step `i`:
    - Calculate candidates: `n2 = ugly[p2]*2`, `n3 = ugly[p3]*3`, `n5 = ugly[p5]*5`.
    - `next_val = min(n2, n3, n5)`.
    - `ugly[i] = next_val`.
    - If `next_val == n2`, increment `p2`.
    - If `next_val == n3`, increment `p3`.
    - If `next_val == n5`, increment `p5`.
    - Note: We use `if` (not `else if`) to handle duplicates (e.g., `2*3` and `3*2` both give 6). Incremeting both pointers skips the duplicate.

### Complexity / Độ phức tạp
- **Time**: O(N) - Generate N numbers.
- **Space**: O(N) - Store N numbers.

---

## Analysis / Phân tích

### Approach: DP with Pointers

**Algorithm**:
1.  Array `ugly` size `n`.
2.  `ugly[0] = 1`.
3.  Loop `i` from 1 to `n-1`.
4.  Compute min and update pointers.
5.  Return `ugly[n-1]`.

---

## Edge Cases / Các trường hợp biên
1.  **n=1**: Return 1.
2.  **Large n**: Fits in Integer? 1690th ugly number is < 2^31 - 1. (Max is around 2*10^9).

Answer fits in 32-bit signed integer.
